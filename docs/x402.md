# x402 Metered Trust API

Nocksperimental can charge micro-`$NOCK` for *consuming* its trust/verification
endpoints using the [x402](https://github.com/coinbase/x402) agentic-payments
protocol, settled on Nockchain via
[zkvesl/x402-nockchain](https://github.com/zkvesl/x402-nockchain). Producing
evidence stays free; consuming verification at scale is metered.

It is **off by default** — a dark-launched feature you flip on with one env var.

## How it works

Nocksperimental acts as an x402 **resource server**:

1. An unpaid request to a metered endpoint returns `402 Payment Required` with a
   `PaymentRequirements` body (paying to the project wallet) and a Bazaar
   discovery block.
2. The client signs an `Authorization` and retries with a `PAYMENT-SIGNATURE`
   header (base64-encoded `PaymentPayload`; `X-PAYMENT` is also accepted).
3. The verifier checks the payment; on success the endpoint returns its normal
   body plus an `X-PAYMENT-RESPONSE` receipt header.

Verification is **pluggable** — the single swap-to-real seam:

- **Stub** (default): offline checks — `x402Version`/scheme/network, recipient
  `== payTo`, amount `>= price`, validity window, and replay (nonce, recorded in
  KV). No on-chain settlement; responses are marked `mode: "stub"`.
- **Facilitator** (set `NOCKS_X402_FACILITATOR_URL`): delegates to a real
  `x402-nockchain-facilitator` `/verify`. No route changes required to switch.

## Metered endpoints and default prices

| Endpoint | Default price (nicks) |
|---|---|
| `GET /api/trust/badges/verify` | 1000 |
| `GET /api/reports/generated/verify` | 1000 |
| `GET /api/fakenet/evidence/verify` | 1000 |
| `GET /api/workspaces/evidence/verify` | 1000 |
| `GET /api/trust/compute-benchmarks/{profileId}` | 10000 |
| `GET /api/trust/token-compatibility/{reportId}` | 10000 |
| `GET /api/invariants/packs/verify` | 1000 |
| `GET /api/nockchain/drift-status/attestation` | 10000 |

The free `GET /api/nockchain/drift-status` snapshot and `GET /api/invariants` catalog stay unmetered (watch boards / discovery, not authority). Only the signed drift-status *attestation* and the invariant-pack *verifier* are metered, mirroring the existing verify/detail tiers (1000 for verification, 10000 for premium signed/detail surfaces).

**Free:** producing evidence (`*/submit`), all list/registry/feed endpoints,
`.well-known`, `openapi.json`, plus a free daily allowance (default 5/day per
client) on metered endpoints so docs and agents can sample. If a valid payment
is present it is charged and the allowance is untouched; only unpaid/invalid
requests draw down the allowance.

## Configuration

| Env var | Default | Meaning |
|---|---|---|
| `NOCKS_X402_ENABLED` | `false` | Master on-switch. Off = routes behave normally (no paywall). |
| `NOCKS_X402_PAY_TO` | `532AxMqc…DqaXQ` | Revenue wallet (the x402 `payTo`). |
| `NOCKS_X402_NETWORK` | `nockchain:fakenet` | CAIP-2 network id. Use `nockchain:mainnet` for real value. |
| `NOCKS_X402_FACILITATOR_URL` | _(unset)_ | A real facilitator URL switches the verifier from stub to facilitator. |
| `NOCKS_X402_FREE_ALLOWANCE_PER_DAY` | `5` | Free metered calls per client per day. |
| `NOCKS_X402_CLOCK_SKEW_SECONDS` | `30` | Validity-window tolerance. |
| `NOCKS_X402_MAX_TIMEOUT_SECONDS` | `120` | Advertised settlement timeout. |

## Discovery

The metered surface is advertised for agents and the VESL Bazaar:

- `/.well-known/nocksperimental.json` — an `x402` block (enabled flag, network,
  `payTo`, per-resource prices, payment headers) plus the
  `x402-metered-trust-api` capability.
- `/openapi.json` — a `402` response documented on each metered path.

## Going live (checklist)

1. Create the receipts KV namespace:
   `npx wrangler kv namespace create NOCKS_X402_RECEIPTS`
2. Add it to `wrangler.jsonc` `kv_namespaces` with the returned id:
   ```json
   { "binding": "NOCKS_X402_RECEIPTS", "id": "<id from step 1>" }
   ```
   Without this binding, receipts / replay protection / allowance fall back to
   per-isolate memory (fine for a disabled feature, not for production metering).
3. Set `NOCKS_X402_PAY_TO` if overriding the default wallet, and — when ready for
   real on-chain settlement — `NOCKS_X402_FACILITATOR_URL`.
4. Flip on: set `NOCKS_X402_ENABLED=true` and redeploy.

## Testing

`npm run test:x402` runs the full suite: config, pricing, header codec, stub
verifier (13 adversarial cases), the 402 → pay → serve meter cycle, facilitator
mode (mocked `fetch`), route gating, and discovery advertising.

## Running a facilitator (provisioning)

The default Stub verifier needs nothing. For real on-chain settlement, run a
[`x402-nockchain-facilitator`](https://github.com/zkvesl/x402-nockchain) and set
`NOCKS_X402_FACILITATOR_URL` — the verifier then switches from stub to real with
no code changes. Building the facilitator locally requires **four sibling repos**
checked out next to each other:

```bash
cd ~
git clone https://github.com/zkvesl/x402-nockchain.git
git clone https://github.com/zkvesl/vesl-core.git      # provides crates/nockchain-client-rs
git clone https://github.com/zkvesl/vesl-wallet.git    # provides crates/vesl-signing
git clone https://github.com/nockchain/nockchain.git   # pin to the SHA x402-nockchain targets
cd x402-nockchain
cargo metadata --no-deps                               # resolves once all four siblings exist
cargo build --release -p x402-nockchain-facilitator
```

**Pinning caveat:** building `nockchain` at HEAD currently fails with a
`libsqlite3-sys` `links` conflict — HEAD's `nockapp` links a different sqlite
version than the workspace expects. Check out `nockchain` (and likely
`vesl-core` / `vesl-wallet`) at the commits x402-nockchain was developed against
— ask the maintainer or consult the repo's CHANGELOG / `Cargo.lock` — then
rebuild.
